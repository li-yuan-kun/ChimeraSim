# Agent: 包结构治理与集成 Agent

## 1. 负责模块
- `chiplet_hetero_sim/**`
- 顶层同名目录：`sim/`, `platform/`, `workload/`, `mapping/`, `runtime/`, `reporting/`
- 文档：`chimerasim_requirements_design_v0_4.md`

## 2. 模块目标
- 统一“可运行代码路径”与“包发布路径”，避免重复实现和导入歧义。

## 3. 当前已完成内容
- 已存在 `chiplet_hetero_sim` 包骨架，目录与设计文档结构基本一致。
- 顶层目录已有一套可运行的最小实现链路，可驱动 `run_sim.py`。

## 4. 当前未完成内容
- 尚未确定单一真实源码来源（source of truth）。
- 包内实现大多是 stub，未与顶层实现同步。
- 缺少迁移策略与兼容层（例如 `run_sim.py` 的导入路径）。

## 5. 发现的问题
- 代码组织存在“双实现分叉”：同名模块在顶层和包内同时存在，且完成度差异大。
- 设计文档宣称的目录为 `chiplet_hetero_sim/*`，但实际执行入口依赖顶层目录，文档与实现不一致。
- 若未来发布为 pip 包，当前入口脚本和导入路径将不可复用。

## 6. 下一阶段任务清单
- [ ] 在技术评审中确定源码主路径（建议 `chiplet_hetero_sim/*` 作为唯一主路径）。
- [ ] 制定迁移计划：将顶层实现按模块迁入包内，并保持行为一致。
- [ ] 为过渡期添加兼容导入层（顶层模块仅转发到包内实现），避免一次性破坏入口。
- [ ] 更新 `run_sim.py` 使用包路径导入，并新增 `python -m chiplet_hetero_sim...` 风格入口。
- [ ] 在阶段文档中维护“迁移完成度看板”（模块级百分比与负责人）。

## 7. 优先级建议
- P0: 确定单一源码路径 + 迁移策略
- P1: 导入兼容层与入口改造
- P2: 清理遗留目录

## 8. 依赖与协作关系
- 依赖所有功能 Agent 在迁移中保持接口稳定。
- 输出交付给文档/发布流程，减少后续维护成本。

## 9. 备注
- “建议以包路径为主”是依据现有设计文档与工程可发布性推断。
